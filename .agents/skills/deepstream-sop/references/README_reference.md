# DeepStream SOP Inference Microservice

A GPU-accelerated FastAPI microservice for **SOP (Standard Operating Procedure) compliance
monitoring** on industrial video. It combines DeepStream GEBD/DDM event-boundary chunking,
embedded vLLM action classification (Cosmos Reason 1/2), SOP sequence/step checking, an
OpenAI-compatible SSE chat-completions API, file / RTSP / Basler-camera inputs, and optional
Kafka publishing.

> Generated by the **`deepstream-sop`** coding-agent skill. To customize, extend, debug, or
> measure latency for this pipeline, run that skill — its `SKILL.md` + `references/` document
> every component, critical rule, env var, and verification step in detail. Upstream blueprint:
> <https://github.com/NVIDIA/sop-monitoring-blueprints>.

## Layout

- `nvds_action_detector/api_server.py` — FastAPI endpoints and SSE streaming.
- `nvds_action_detector/ds_sop_process.py` — SOP process manager and four-stage request pipeline.
- `nvds_action_detector/ds_3d_action_pipeline.py` — DeepStream/Triton DDM pipeline.
- `nvds_action_detector/vllm_inference.py` — embedded vLLM inference.
- `nvds_action_detector/sop_step_checker.py` — SOP sequence checker.
- `docker/Docker.build` — image with DeepStream, Pylon, DDM, protobuf, and the custom postprocess build.
- `deploy/compose.yaml` — microservice + Kafka deployment.
- `tests/test_api_endpoints.py` — API endpoint regression suite.

## Build

Place the Pylon SDK tarball (a commercial, license-gated download from Basler after
registration; not redistributed here) at:

```bash
binaries/pylon-25.10.2_linux-x86_64_setup.tar.gz
```

The `binaries/` directory is tracked via `binaries/.gitkeep` so the build's bind mount
resolves on a fresh clone; the SDK tarball itself is git-ignored. If the tarball is
missing, the build stops with instructions. To accept the Basler license terms and let
the build download it from the Basler CDN, pass `ALLOW_PYLON_CDN_DOWNLOAD=1`.

Configure model paths in `deploy/.env` (from `deploy/.env.example`) — `VLLM_MODEL_PATH`
and `DDM_MODEL_PATH` are **required** (no defaults; the service fails fast if unset) —
then build:

```bash
docker compose -f deploy/compose.yaml build nvds-action-sop

# Or, to accept the Basler license terms and auto-download the Pylon SDK:
ALLOW_PYLON_CDN_DOWNLOAD=1 docker compose -f deploy/compose.yaml build nvds-action-sop
```

## Run

```bash
docker compose --env-file deploy/.env -f deploy/compose.yaml up -d
# Model load takes several minutes on first start — poll until ready:
until curl -fs http://localhost:8300/v1/ready >/dev/null; do sleep 5; done
```

API-only validation without GPU models:

```bash
API_DUMMY_TEST=true PYTHONPATH=. python3 -m nvds_action_detector.api_server
TEST_VIDEO_PATH=configs/actions.json python3 tests/test_api_endpoints.py
```

## Test the inference API

OpenAI-compatible API on `http://localhost:8300`
(`/v1/live`, `/v1/ready`, `/v1/models`, `/v1/files`, `/v1/chat/completions`, `/v1/metrics`).

> The VLM prompt is loaded from `VLM_PROMPT_PATH`. Do **not** add a `{"type": "text"}` content
> item — it overrides the config-file prompt and breaks SOP step classification.

### Video file

Upload the clip, then stream inference on the returned `file_id`:

```bash
FILE_ID=$(curl -s -X POST http://localhost:8300/v1/files \
  -F "file=@test_video.mp4" -F "purpose=inference" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")

curl -N -X POST http://localhost:8300/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"ds_sop_model","stream":true,
       "messages":[{"role":"user","content":[{"type":"input_video","file_id":"'"$FILE_ID"'"}]}],
       "chunking_options":{"algorithm":"ddm-net","threshold":0.8,"min_length_sec":1.0,"max_length_sec":10.0}}'
```

### RTSP stream

```bash
curl -N -X POST http://localhost:8300/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"ds_sop_model","stream":true,
       "messages":[{"role":"user","content":[{"type":"video_url","video_url":{"url":"rtsp://<host>:8554/stream"}}]}],
       "chunking_options":{"algorithm":"ddm-net","threshold":0.8}}'
```

### Basler camera

Live cameras require `"stream": true`; use a small `max_length_sec` (≤ 5 s) for responsive streaming:

```bash
curl -N -X POST http://localhost:8300/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"ds_sop_model","stream":true,
       "messages":[{"role":"user","content":[{"type":"input_camera","input_camera":{
         "camera_id":"40748152","camera_vendor":"Basler","camera_format":"RGB",
         "camera_width":1280,"camera_height":720,"camera_fps_num":30,"camera_fps_den":1}}]}],
       "chunking_options":{"algorithm":"ddm-net","threshold":0.8,"max_length_sec":2.0}}'
```
